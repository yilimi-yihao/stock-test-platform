sql-tool

A 股本地数据库维护工具，支持 GUI 管理、Tushare 导入、SQLite 存储与本地 HTTP API。

快速开始

1. 安装依赖

pip install -r requirements.txt

2. 配置 Token

编辑 config/settings.json，填入 Tushare Token。

3. 检查接口能力

python main.py detect

4. 启动 GUI

python main.py gui

推荐使用顺序

  1. 先检查 token 是否具备需要的接口权限。
  2. 先导入少量股票验证链路。
  3. 确认数据库状态正常后，再做全量导入或增量更新。
  4. 需要给其他工程供数时，在 GUI 的“API 服务”页或 CLI 中启动本地 API。

GUI 页面

页签说明
数据管理导入、更新、清库、刷新统计
API 服务配置 host/port、启动/停止 API、检查状态、打开文档
数据库状态查看数据库路径、大小、表计数、更新时间、日期范围
接口检测检测当前 token 对核心接口的可用性
数据浏览查看股票最近日线与财务摘要
帮助打开本地帮助页与 API 文档入口

本地 API

默认地址：http://127.0.0.1:8000
Swagger 文档：/docs
健康检查：/health 
python main.py api
python main.py api --host 127.0.0.1 --port 8011

GUI 启动 API 后，可直接点击“打开 API 文档”。

常用命令

python main.py stats
python main.py detect
python main.py import --limit 100
python main.py import --all
python main.py update
python main.py api

常见问题

为什么接口检测全失败？

通常是 token 权限不足、token 无效或网络不可达。

打开 API 文档没内容怎么办？

先确认本地 API 已启动，并能正常访问 /health。

数据库文件在哪？

默认是 data/a_share.db，也可以在配置文件中修改。

提示：如果 GUI 里显示“端口已占用”，说明该地址可能已有其他服务运行，先更换端口或停止已有进程。
